# Penetration test — Statement of Work (2026 Q2) — template

**Status:** **Template only** — fill when a **V2** third-party vendor is selected and funded. **Do not** embed production secrets in this repository.

**Independent quality assessments must not** treat absence of an executed third-party SoW as a **V1** deficit.

## 1. Objective

Authorize a **time-boxed** technical assessment of ArchLucid’s **staging** attack surface aligned to OWASP ASVS-style depth, without destructive mutation of production.

## 2. Scope

| In scope | Out of scope (unless explicitly added) |
|----------|------------------------------------------|
| Public HTTPS **v1** API (`/v1/*`, OpenAPI contract) | Customer-owned IdP misconfiguration |
| Operator UI (`archlucid-ui`) against staging tenant | Physical / social engineering |
| Multi-tenant isolation: SQL **RLS** + `SESSION_CONTEXT` + documented **break-glass** paths | Third-party SaaS outside [subprocessors](../../go-to-market/SUBPROCESSORS.md) |
| **JWT** (Entra) + **API key** automation paths | Denial-of-service against shared CI |
| **Billing:** `BillingStripeWebhookController` (signing secret, replay, tenant binding) | Destructive production testing |
| **LLM:** outbound Azure OpenAI path — prompt boundary, **redaction**, content safety | |

## 3. Methodology

- OWASP ASVS-aligned testing; **no** destructive data mutation on production.
- Credential rotation after dedicated test accounts are used; **break-glass** SQL principals forbidden unless pre-approved in writing.

## 4. Deliverables

1. Executive summary (risk-ranked).
2. Detailed findings with reproduction, impact, and remediation guidance.
3. Negotiated re-test window for critical findings.
